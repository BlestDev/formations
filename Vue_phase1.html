Pourquoi cet environnement ?

Avant d'écrire une seule ligne de code, tu as besoin d'un atelier bien équipé. L'environnement de développement, c'est exactement ça : l'ensemble des outils qui te permettent d'écrire, tester, versionner et déployer ton code.

1. VS Code — ton éditeur

VS Code est l'éditeur de code le plus utilisé au monde. Il est gratuit, rapide, et s'étend via des extensions. Télécharge-le sur code.visualstudio.com.

Extensions indispensables à installer tout de suite :

  • Prettier — formate ton code automatiquement à la sauvegarde
  • ESLint — détecte les erreurs JS avant d'exécuter le code
  • GitLens — visualise l'historique Git dans l'éditeur
  • Vue - Official (ou React Developer Tools pour React)
  • Thunder Client — tester des API directement dans VS Code
Configuration Prettier Crée un fichier .prettierrc à la racine de chaque projet avec : {"semi": true, "singleQuote": true, "tabWidth": 2}. Active "Format on Save" dans les paramètres VS Code (Ctrl+Shift+P → "Open Settings" → chercher "Format On Save").

2. Node.js — le moteur JavaScript

Node.js permet d'exécuter du JavaScript en dehors du navigateur. Il inclut npm (gestionnaire de paquets). Télécharge la version LTS sur nodejs.org.

Terminal — vérifier l'installation
# Après installation, vérifier les versions
node --version    # doit afficher v20.x.x ou supérieur
npm --version     # doit afficher 10.x.x ou supérieur

3. Le terminal — ton outil de commande

Le terminal (ou invite de commande) est l'interface texte pour piloter ton ordinateur. Sur Mac/Linux : Terminal. Sur Windows : utilise Git Bash (inclus avec Git) ou le terminal intégré de VS Code.

Commandes essentielles à connaître
# Navigation
pwd               # affiche le répertoire courant
ls                # liste les fichiers (dir sur Windows)
cd nom-dossier    # entrer dans un dossier
cd ..             # remonter d'un niveau
cd ~              # aller dans le dossier personnel

# Fichiers & dossiers
mkdir mon-projet  # créer un dossier
touch index.html  # créer un fichier vide (Mac/Linux)
rm fichier.txt    # supprimer un fichier (⚠ sans corbeille)
cp a.txt b.txt    # copier
mv a.txt dossier/ # déplacer / renommer

# Utile
clear             # vider le terminal
Ctrl+C            # arrêter un processus en cours
Piège courant La commande rm -rf supprime définitivement des dossiers entiers sans confirmation. Ne l'utilise jamais sans être certain de ce que tu supprimes.

Comprendre Git

Git est un système de contrôle de version. Il enregistre chaque modification de ton code dans des "commits" — des snapshots. Si tu casses quelque chose, tu peux revenir en arrière. C'est indispensable, même seul.

GitHub est une plateforme en ligne pour stocker tes dépôts Git. Crée un compte gratuit sur github.com.

Les 4 commandes Git du quotidien

Workflow Git de base
# 1. Initialiser un dépôt (une seule fois par projet)
git init

# 2. Voir l'état de tes fichiers
git status

# 3. Ajouter des fichiers à la "zone de staging"
git add index.html    # un seul fichier
git add .             # tous les fichiers modifiés

# 4. Créer un commit (un snapshot)
git commit -m "feat: ajoute la page d'accueil"

# 5. Envoyer sur GitHub
git push origin main

Connecter un projet local à GitHub

Lier un repo local à GitHub
# Sur GitHub : créer un nouveau repo (sans README)
# Puis dans ton terminal :

git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/TON-USER/TON-REPO.git
git push -u origin main

Convention de nommage des commits

Format recommandé (Conventional Commits)
feat: ajoute le formulaire de connexion
fix: corrige l'affichage sur mobile
style: reformate le CSS
docs: met à jour le README
refactor: simplifie la fonction de tri
Fichier .gitignore Crée un fichier .gitignore pour exclure les fichiers inutiles. Au minimum : node_modules/ et .env. GitHub propose des templates selon le projet.

HTML sémantique

Le HTML sémantique utilise des balises qui décrivent le rôle du contenu, pas juste son apparence. Ça aide les moteurs de recherche, les lecteurs d'écran, et tes collègues.

Structure d'une page HTML moderne

index.html — squelette complet
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Ma page</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <header>              <!-- en-tête du site -->
    <nav>               <!-- navigation principale -->
      <a href="/">Accueil</a>
    </nav>
  </header>

  <main>                <!-- contenu principal (1 seul par page) -->
    <article>           <!-- contenu autonome (article, carte) -->
      <h1>Titre principal</h1>
      <p>Introduction...</p>
      <section>          <!-- section thématique -->
        <h2>Sous-titre</h2>
      </section>
    </article>

    <aside>             <!-- contenu secondaire (sidebar) -->
    </aside>
  </main>

  <footer>              <!-- pied de page -->
    <p>© 2025</p>
  </footer>

  <script src="app.js"></script>  <!-- JS à la fin du body -->
</body>
</html>

Balises à connaître absolument

Balises courantes
<!-- Titres (hiérarchie h1 → h6, un seul h1 par page) -->
<h1>Titre principal</h1>
<h2>Sous-titre</h2>

<!-- Listes -->
<ul><li>Non ordonnée</li></ul>
<ol><li>Ordonnée</li></ol>

<!-- Image (toujours un alt !) -->
<img src="photo.jpg" alt="Description de l'image">

<!-- Formulaire -->
<form>
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">Envoyer</button>
</form>

Fonctions fléchées

fonctions.js
// Fonction classique
function saluer(nom) {
  return `Bonjour ${nom}`;
}

// Fonction fléchée (équivalent)
const saluer = (nom) => `Bonjour ${nom}`;

// Paramètre par défaut
const saluer = (nom = 'visiteur') => `Bonjour ${nom}`;

// Plusieurs paramètres
const additionner = (a, b) => a + b;

// Corps de fonction multi-lignes
const calculer = (prix, tva) => {
  const total = prix * (1 + tva);
  return Math.round(total * 100) / 100;
};

Async / Await & fetch()
Appels API · Promises · Gestion d'erreurs
JavaScript asynchrone

JavaScript est non-bloquant : il peut lancer une opération longue (appel réseau, lecture de fichier) et continuer à s'exécuter pendant ce temps. async/await est la façon moderne de gérer ça.

Promise → async/await

async.js
// Version Promise (ancienne façon)
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// Version async/await (façon moderne — préférer)
async function chargerDonnees() {
  try {
    const response = await fetch('https://api.example.com/data');

    // Vérifier que la requête a réussi
    if (!response.ok) {
      throw new Error(`Erreur HTTP: ${response.status}`);
    }

    const data = await response.json();
    console.log(data);
    return data;

  } catch (erreur) {
    console.error('Erreur :', erreur.message);
  }
}

Exemple complet : afficher des posts

posts.js — appel API réel
// JSONPlaceholder : API de test gratuite
const URL_API = 'https://jsonplaceholder.typicode.com';

async function afficherPosts() {
  // 1. Montrer un indicateur de chargement
  const conteneur = document.querySelector('#posts');
  conteneur.innerHTML = '<p>Chargement...</p>';

  try {
    // 2. Appel API
    const response = await fetch(`${URL_API}/posts?_limit=5`);
    const posts = await response.json();

    // 3. Afficher les données
    conteneur.innerHTML = posts
      .map(post => `
        <article>
          <h3>${post.title}</h3>
          <p>${post.body}</p>
        </article>
      `)
      .join('');

  } catch (erreur) {
    // 4. Gérer l'erreur
    conteneur.innerHTML = `<p>Erreur : ${erreur.message}</p>`;
  }
}

// Appeler quand la page est prête
document.addEventListener('DOMContentLoaded', afficherPosts);
JSONPlaceholder — API de test jsonplaceholder.typicode.com fournit des endpoints gratuits pour tester : /posts, /users, /photos, /todos. Parfait pour s'entraîner sans créer de backend.

Description du projet

Tu vas construire une vraie application météo qui utilise Open-Meteo (API gratuite, sans clé requise). L'utilisateur saisit une ville, l'app affiche la météo actuelle et les prévisions.

Fonctionnalités requises

  • Champ de recherche de ville
  • Affichage : température, condition météo, humidité, vent
  • Icône météo selon la condition
  • Prévisions sur 5 jours (optionnel)
  • Design responsive et soigné
  • Gestion des erreurs (ville introuvable)

Architecture recommandée

Structure du projet

├── index.html
├── style.css
├── js/
│   ├── api.js       ← fonctions d'appel API

Étape 1 — Géolocalisation de la ville

js/api.js
// Open-Meteo Geocoding API — convertir ville → coordonnées
export async function rechercherVille(nomVille) {
  const url = `https://geocoding-api.open-meteo.com/v1/search?name=${encodeURIComponent(nomVille)}&count=1&language=fr`;
  const response = await fetch(url);
  const data = await response.json();

  if (!data.results || data.results.length === 0) {
    throw new Error('Ville introuvable');
  }

  return data.results[0]; // { name, latitude, longitude, country }
}

Étape 2 — Appel météo

js/api.js (suite)
export async function recupererMeteo(latitude, longitude) {
  const params = new URLSearchParams({
    latitude, longitude,
    current: 'temperature_2m,relative_humidity_2m,wind_speed_10m,weather_code',
    daily: 'temperature_2m_max,temperature_2m_min,weather_code',
    timezone: 'auto',
    forecast_days: 5
  });

  const url = `https://api.open-meteo.com/v1/forecast?${params}`;
  const response = await fetch(url);
  return await response.json();
}

// Convertir le code météo en description
export function codeVersDescription(code) {
  const codes = {
    0: 'Ciel dégagé ☀️',  1: 'Peu nuageux 🌤️',
    2: 'Partiellement nuageux ⛅', 3: 'Couvert ☁️',
    61: 'Pluie légère 🌧️', 63: 'Pluie modérée 🌧️',
    71: 'Neige 🌨️', 95: 'Orage ⛈️'
  };
  return codes[code] || 'Conditions inconnues 🌡️';
}

Étape 3 — Logique principale

js/main.js
import { rechercherVille, recupererMeteo, codeVersDescription }
  from './api.js';

const formulaire = document.querySelector('#recherche');
const resultat = document.querySelector('#resultat');

formulaire.addEventListener('submit', async (e) => {
  e.preventDefault();
  const ville = formulaire.querySelector('input').value.trim();
  if (!ville) return;

  resultat.innerHTML = '<p class="chargement">Recherche...</p>';

  try {
    const lieu = await rechercherVille(ville);
    const meteo = await recupererMeteo(lieu.latitude, lieu.longitude);
    const actuelle = meteo.current;

    resultat.innerHTML = `
      <div class="meteo-carte">
        <h2>${lieu.name}, ${lieu.country}</h2>
        <div class="temperature">${Math.round(actuelle.temperature_2m)}°C</div>
        <p>${codeVersDescription(actuelle.weather_code)}</p>
        <div class="details">
          <span>💧 ${actuelle.relative_humidity_2m}%</span>
          <span>💨 ${actuelle.wind_speed_10m} km/h</span>
        </div>
      </div>
    `;
  } catch (err) {
    resultat.innerHTML = `<p class="erreur">❌ ${err.message}</p>`;
  }
});
