Le problème que Vue résout

Avec du JS pur, synchroniser l'interface et les données devient vite un cauchemar. Tu dois manuellement trouver les éléments DOM et les mettre à jour à chaque changement.

Problème avec JS purJS
// Avec JS pur : beaucoup de code pour peu de résultat
let compteur = 0;
const btn = document.querySelector('#btn');
const affichage = document.querySelector('#affichage');

btn.addEventListener('click', () => {
  compteur++;
  affichage.textContent = compteur; // ← MAJ manuelle du DOM
});
// Et si tu as 50 endroits à mettre à jour ? 😰
Solution avec Vue 3Vue
<script setup>
import { ref } from 'vue'
const compteur = ref(0)
</script>

<template>
  <!-- Vue met à jour le DOM automatiquement -->
  <button @click="compteur++">Cliqué {{ compteur }} fois</button>
</template>

src/components/MonComposant.vueVue
<!-- 1. Le template : le HTML du composant -->
<template>
  <div class="ma-carte">
    <h2>{{ titre }}</h2>
    <p>Compteur : {{ compteur }}</p>
    <button @click="incrementer">+1</button>

import { ref } from 'vue'

  background: white;
  padding: 1rem;
  border-radius: 8px;

</style>
ref().value — la règle d'or Dans <script setup>, pour lire ou modifier un ref, utilise compteur.value. Mais dans le <template>, Vue "déballe" automatiquement : écris juste {{ compteur }} sans .value.

ref, reactive & computed
Les trois outils de la réactivité Vue 3
La réactivité en détail

ref() — valeurs primitives

Utilise ref() pour les valeurs simples : nombres, chaînes, booléens, et même les tableaux.

ref() — exemplesVue
import { ref } from 'vue'

// Valeurs primitives
const nom = ref('Alice')
const age = ref(25)
const estConnecte = ref(false)
const taches = ref([])   // tableau vide

// Modifier (toujours .value dans le script)
nom.value = 'Bob'
age.value++
estConnecte.value = true
taches.value.push({ id: 1, texte: 'Apprendre Vue' })

reactive() — objets complexes

Utilise reactive() pour les objets. Pas besoin de .value, tu accèdes directement aux propriétés.

reactive() — exemplesVue
import { reactive } from 'vue'

const utilisateur = reactive({
  nom: 'Alice',
  age: 25,
  adresse: {
    ville: 'Paris'
  }
})

// Modifier directement (pas de .value)
utilisateur.nom = 'Bob'
utilisateur.age++
utilisateur.adresse.ville = 'Lyon'

// Dans le template :
// <p>{{ utilisateur.nom }} habite à {{ utilisateur.adresse.ville }}</p>

computed() — valeurs dérivées

Un computed se recalcule automatiquement quand ses dépendances changent. Ne jamais modifier un computed directement.

computed() — exemple completVue
import { ref, computed } from 'vue'

const taches = ref([
  { id: 1, texte: 'Apprendre Vue', fait: true },
  { id: 2, texte: 'Créer un projet', fait: false },
  { id: 3, texte: 'Déployer', fait: false }
])

// Se recalcule automatiquement quand taches.value change
const nombreFaites = computed(() =>
  taches.value.filter(t => t.fait).length
)

const tachesRestantes = computed(() =>
  taches.value.filter(t => !t.fait)
)

const progression = computed(() => {
  if (taches.value.length === 0) return 0
  return Math.round(nombreFaites.value / taches.value.length * 100)
})

// Dans le template :
// <p>{{ nombreFaites }} / {{ taches.length }} ({{ progression }}%)</p>
Règle simple : ref ou reactive ? Commence par tout mettre dans ref(). C'est plus simple et cohérent. Utilise reactive() uniquement pour des objets avec plusieurs propriétés liées (ex : un formulaire avec 5 champs).

Directives du template
v-if · v-for · v-bind · v-model · v-on
Les directives Vue

Les directives sont des attributs spéciaux préfixés par v- qui donnent des instructions à Vue sur comment rendre le DOM.

Toutes les directives essentiellesVue
<template>
  <!-- v-if / v-else-if / v-else : affichage conditionnel -->
  <p v-if="estConnecte">Bienvenue !</p>
  <p v-else>Veuillez vous connecter</p>

  <!-- v-show : masque via CSS (reste dans le DOM) -->
  <div v-show="chargement">Chargement...</div>

  <!-- v-for : boucle (toujours avec :key !) -->
  <ul>
    <li v-for="tache in taches" :key="tache.id">
      {{ tache.texte }}
    </li>
  </ul>

  <!-- v-bind (raccourci :) : lier un attribut à une donnée -->
  <img :src="imageUrl" :alt="imageAlt">
  <button :disabled="estChargement">Envoyer</button>
  <div :class="{ 'actif': estActif, 'erreur': aUneErreur }"></div>

  <!-- v-on (raccourci @) : écouter des événements -->
  <button @click="soumettre">Envoyer</button>
  <input @keyup.enter="chercher">
  <form @submit.prevent="valider"></form>

  <!-- v-model : liaison bidirectionnelle (input ↔ donnée) -->
  <input v-model="recherche" placeholder="Chercher...">
  <textarea v-model="message"></textarea>
  <input type="checkbox" v-model="accepte">
  <select v-model="categorie">
    <option value="vue">Vue</option>
  </select>
</template>
v-for sans :key = bug Toujours ajouter :key sur un v-for. C'est ce qui permet à Vue de savoir quel élément mettre à jour. Utilise un identifiant unique (id de la donnée), jamais l'index du tableau.

Exemple : liste de tâches avec interactions

Taches.vue — composant completVue
<script setup>
import { ref, computed } from 'vue'

const nouvelleTache = ref('')
const taches = ref([
  { id: 1, texte: 'Apprendre Vue 3', fait: false }
])

const restantes = computed(() => taches.value.filter(t => !t.fait).length)

function ajouterTache() {
  if (!nouvelleTache.value.trim()) return
  taches.value.push({
    id: Date.now(),
    texte: nouvelleTache.value,
    fait: false
  })
  nouvelleTache.value = ''
}

function supprimerTache(id) {
  taches.value = taches.value.filter(t => t.id !== id)
}
</script>

<template>
  <div>
    <h2>Tâches ({{ restantes }} restantes)</h2>

    <form @submit.prevent="ajouterTache">
      <input v-model="nouvelleTache" placeholder="Nouvelle tâche...">
      <button type="submit">Ajouter</button>
    </form>

    <p v-if="taches.length === 0">Aucune tâche 🎉</p>

    <ul v-else>
      <li v-for="tache in taches" :key="tache.id">
        <input type="checkbox" v-model="tache.fait">
        <span :class="{ 'fait': tache.fait }">{{ tache.texte }}</span>
        <button @click="supprimerTache(tache.id)">✕</button>
      </li>
    </ul>
  </div>
</template>

<script setup>
import { reactive } from 'vue'

// Déclarer les événements que ce composant peut émettre
const emit = defineEmits(['soumettre', 'annuler'])

const formulaire = reactive({
  nom: '',
  email: '',
  message: ''
})

function valider() {
  // Émettre l'événement avec les données du formulaire

</template>

Les slots permettent à un composant parent d'injecter du contenu HTML à l'intérieur d'un composant enfant. C'est comme un "trou" que le parent remplit.

<Carte>
  <template #titre>

Slots vs Props — quand utiliser quoi ? Props = données simples (texte, nombre, booléen). Slots = contenu HTML complexe que le parent décide. Règle : si tu as besoin de passer du HTML structuré avec des composants enfants, utilise un slot.

Routing côté client

Vue Router permet de naviguer entre "pages" sans recharger le navigateur. L'URL change, le composant affiché change, mais c'est toujours la même app.

Configuration des routes

src/router/index.jsJS
import { createRouter, createWebHistory } from 'vue-router'
import AccueilView from '../views/AccueilView.vue'

const routes = [
  // Route simple
  { path: '/', name: 'accueil', component: AccueilView },

  // Lazy loading (chargé seulement à la visite)
  {
    path: '/notes',
    name: 'notes',
    component: () => import('../views/NotesView.vue')
  },

  // Route dynamique (paramètre dans l'URL)
  {
    path: '/notes/:id',
    name: 'note-detail',
    component: () => import('../views/NoteDetailView.vue')
  },

  // Route protégée avec meta
  {
    path: '/profil',
    name: 'profil',
    component: () => import('../views/ProfilView.vue'),
    meta: { requiresAuth: true }
  },

  // Page 404
  { path: '/:pathMatch(.*)*', name: 'not-found',
    component: () => import('../views/NotFoundView.vue') }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// Navigation guard (protéger les routes)
router.beforeEach((to) => {
  const estConnecte = localStorage.getItem('token')
  if (to.meta.requiresAuth && !estConnecte) {
    return { name: 'accueil' }
  }
})

export default router

Pinia — État global
Partager des données entre tous les composants
Le problème que Pinia résout

Quand deux composants "cousins" (pas parent-enfant) ont besoin des mêmes données, passer des props devient impossible. Pinia crée un "magasin" global accessible partout.

Pinia Store ←──── accès direct ────→ N'importe quel composant (état, actions)

Créer un store

src/stores/notes.jsJS
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

// Convention : useXxxStore
export const useNotesStore = defineStore('notes', () => {
  // STATE : les données
  const notes = ref([])
  const chargement = ref(false)

  // GETTERS : données dérivées (comme computed)
  const nombreNotes = computed(() => notes.value.length)
  const notesFavorites = computed(() =>
    notes.value.filter(n => n.favori)
  )

  // ACTIONS : fonctions qui modifient le state
  function ajouterNote(titre, contenu) {
    notes.value.push({
      id: Date.now(),
      titre,
      contenu,
      favori: false,
      creeLe: new Date().toISOString()
    })
  }

  function supprimerNote(id) {
    notes.value = notes.value.filter(n => n.id !== id)
  }

  function basculerFavori(id) {
    const note = notes.value.find(n => n.id === id)
    if (note) note.favori = !note.favori
  }

  return {
    notes, chargement,
    nombreNotes, notesFavorites,
    ajouterNote, supprimerNote, basculerFavori
  }
})

Utiliser le store dans n'importe quel composant

N'importe quel composant .vueVue
<script setup>
import { useNotesStore } from '../stores/notes'

// Accéder au store — une seule instance partagée

</script>

<template>

  <div v-for="note in notesStore.notes" :key="note.id">
    {{ note.titre }}
    <button @click="notesStore.supprimerNote(note.id)">✕</button>
  </div>
</template>

Lifecycle Hooks
Quand exécuter du code pendant la vie d'un composant
1h30 théorie
2h30 pratique
Cycle de vie d'un composant
setup() → composant initialisé ↓ onBeforeMount() → avant insertion dans le DOM ↓ onMounted() → composant dans le DOM ✅ ← charger les données ici ↓ onUpdated() → après chaque mise à jour ↓ onUnmounted() → composant retiré du DOM → nettoyer ici
Lifecycle hooks en pratiqueVue
import { ref, onMounted, onUnmounted, watch } from 'vue'

const donnees = ref(null)
const chargement = ref(true)
let intervalle = null

// ✅ Charger les données une fois le composant monté
onMounted(async () => {
  try {
    const res = await fetch('/api/data')
    donnees.value = await res.json()
  } finally {
    chargement.value = false
  }

  // Démarrer un intervalle
  intervalle = setInterval(() => rafraichir(), 30000)
})

// 🧹 Nettoyer quand le composant disparaît
onUnmounted(() => {
  clearInterval(intervalle)
})

// 👁️ Réagir aux changements d'une valeur
const route = useRoute()

Composable — logique réutilisable

Un composable est une fonction qui encapsule de la logique réactive (comme un hook React). Convention : préfixe use.

src/composables/useFetch.jsJS
import { ref } from 'vue'

export function useFetch(url) {
  const donnees = ref(null)
  const chargement = ref(true)
  const erreur = ref(null)

  async function charger() {
    chargement.value = true
    erreur.value = null
    try {
      const res = await fetch(url)
      if (!res.ok) throw new Error(`HTTP ${res.status}`)
      donnees.value = await res.json()
    } catch (e) {
      erreur.value = e.message
    } finally {
      chargement.value = false
    }
  }

  charger()  // lancer au démarrage

  return { donnees, chargement, erreur, recharger: charger }
}

Store Pinia complet

src/stores/notes.jsJS
import { defineStore } from 'pinia'
import { ref, computed, watch } from 'vue'

export const useNotesStore = defineStore('notes', () => {
  // Charger depuis localStorage au démarrage
  const notesEnregistrees = localStorage.getItem('mes-notes')
  const notes = ref(notesEnregistrees ? JSON.parse(notesEnregistrees) : [])
  const filtreActif = ref('toutes') // 'toutes' | 'favorites'
  const recherche = ref('')

  // Sauvegarder automatiquement
  watch(notes, (val) => {
    localStorage.setItem('mes-notes', JSON.stringify(val))
  }, { deep: true })

  const notesFiltrees = computed(() => {
    let resultat = notes.value
    if (filtreActif.value === 'favorites')
      resultat = resultat.filter(n => n.favori)
    if (recherche.value)
      resultat = resultat.filter(n =>
        n.titre.toLowerCase().includes(recherche.value.toLowerCase())
      )
    return resultat.sort((a, b) => new Date(b.creeLe) - new Date(a.creeLe))
  })

  function trouverParId(id) {
    return notes.value.find(n => n.id === id)
  }

  function ajouterNote(donnees) {
    notes.value.push({
      id: crypto.randomUUID(),
      ...donnees,
      favori: false,
      creeLe: new Date().toISOString()
    })
  }

  function mettreAJour(id, donnees) {
    const idx = notes.value.findIndex(n => n.id === id)
    if (idx >= 0) notes.value[idx] = { ...notes.value[idx], ...donnees }
  }

  function supprimer(id) {
    notes.value = notes.value.filter(n => n.id !== id)
  }

  function basculerFavori(id) {
    const note = trouverParId(id)
    if (note) note.favori = !note.favori
  }

  return {
    notes, filtreActif, recherche, notesFiltrees,
    trouverParId, ajouterNote, mettreAJour, supprimer, basculerFavori
  }
})
